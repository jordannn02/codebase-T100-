# 範例 Prompts

以下範例都使用假代號，避免暴露私人資料。

## 程式影響分析

```text
請用 codebase-T100版 分析 axmt500，如果修改 \[業務人員](xmda002)，可能影響哪些程式、表、欄位與後續流程。
先用 graph 找 caller/callee，再回原始碼確認條件。
```

## 欄位來源追蹤

```text
請追 cxmta41 裡 \[計算結果](xxyy001) 的來源。
輸出需要包含程式、函式、表名、欄位、計算條件，以及哪些地方仍需要 DB 或 runtime 驗證。
```

## 4GL / PER 畫面欄位對照

```text
請查 asft311 的畫面欄位與 4GL 邏輯，找出批號日期相關欄位。
請分成畫面欄位、source 使用點、表欄位、待驗證 SQL。
```

## 標準與客製差異

```text
請比較標準程式與 csub 客製邏輯，列出改寫點、分支條件、可能風險。
不要只列檔名，要回到 source 片段確認。
```
